### 🧠 Multimodal Lie Detection System — Final Year Project
**✅ Completed**

**Problem:** Single-modality deception detection (facial analysis or audio alone) produces unreliable results across different subjects and recording conditions. No single signal source is sufficient for generalization.

**Approach:** Designed a three-modality pipeline that processes EEG signals, facial video, and audio independently, then fuses the outputs into a combined prediction. Each modality has its own preprocessing, feature extraction, and trained classifier — integrated through a Flask backend with a React Native mobile client.

| Modality | Model | Feature Source | Accuracy |
|:---|:---|:---|:---:|
| Video — Facial Expressions | BiLSTM | DeepFace action units via OpenCV | **93%** |
| EEG — Brainwave Signals | XGBoost | Band power (α, β, γ, θ, δ) — Muse 2 | **83%** |
| Audio — Voice Analysis | XGBoost | 96 Librosa features (MFCCs, pitch, spectral) | **88%** |

> Accuracy figures are evaluation results on held-out test data per modality.

**Key Contributions:**
- Implemented EEG band power extraction from raw Muse 2 recordings and trained an XGBoost classifier on the resulting feature vectors
- Built a facial sequence pipeline using OpenCV and DeepFace for frame-level feature extraction, then trained a BiLSTM on temporal sequences
- Designed the 96-feature audio pipeline with Librosa and trained a separate XGBoost classifier on acoustic features
- Developed the Flask REST API for session management, multipart file upload handling, and per-modality inference endpoints
- Integrated all three modalities into the React Native frontend with result aggregation and confidence display

### 📄 DocuMind AI — RAG Document Q&A System
**🔨 In Progress**

**Problem:** Keyword-based document search returns low-relevance results and requires users to manually scan through large files. Standard search does not understand semantic meaning or context.

**Approach:** Built a retrieval-augmented generation (RAG) pipeline that ingests PDF documents, splits them into overlapping chunks, generates embeddings, and indexes them into a vector database. At query time, the system retrieves semantically relevant chunks and passes them as grounded context to an LLM for answer generation.

**Key Contributions:**
- Implemented the document ingestion pipeline: PDF parsing, text chunking with configurable overlap, and embedding generation
- Integrated a vector database for similarity-based retrieval with configurable top-k results
- Designed the prompt construction layer to pass retrieved context alongside user queries for source-grounded responses
- Developed a FastAPI backend exposing document upload and Q&A query endpoints
- Handled edge cases including multi-page PDFs, scanned documents, and long-context truncation

**Outcome:** End-to-end pipeline supporting semantic Q&A over uploaded documents with source-attributed responses.

### 💼 JobSense AI — Resume-to-Job Matching System
**🔨 In Progress**

**Problem:** Comparing resumes against job descriptions manually is time-intensive and inconsistent. There is no structured way to quantify candidate-role fit from unstructured text.

**Approach:** Built an NLP pipeline that parses resumes and job postings, extracts structured entities (skills, experience level, qualifications), and scores candidate-job compatibility using similarity-based matching.

**Key Contributions:**
- Implemented a resume parser that processes unstructured PDF and DOCX text into structured fields: skills, education, experience
- Designed an NLP entity extraction layer to identify skills, job titles, and qualifications from free-form text
- Developed a matching module that scores and ranks candidates against a given job description
- Built a FastAPI backend exposing resume parsing and matching as REST endpoints

**Outcome:** Functional matching pipeline that returns ranked candidate scores with per-field compatibility breakdowns.

### 🚗 Vehicle Detection & Traffic Analytics System
**✅ Completed**

**Problem:** Manual traffic monitoring does not scale for multi-lane or real-time environments. Video-based monitoring requires automated pipelines to detect, track, and analyze vehicle movement across frames.

**Approach:** Built an OpenCV-based detection and tracking pipeline that processes video frame-by-frame, identifies vehicles using a pre-trained YOLO model, tracks them across frames with centroid-based tracking, and aggregates per-frame detections into traffic analytics.

**Key Contributions:**
- Integrated a pre-trained YOLO detection model with an OpenCV video processing loop for frame-level vehicle detection
- Implemented centroid-based tracking to maintain vehicle identity across frames, including basic occlusion handling
- Built an analytics layer to compute vehicle counts, estimated speeds, and lane distribution from tracking output
- Designed the pipeline to handle variable lighting conditions and multi-lane footage

**Outcome:** Functional pipeline for video-based traffic monitoring with per-lane vehicle counts and flow statistics as output.
